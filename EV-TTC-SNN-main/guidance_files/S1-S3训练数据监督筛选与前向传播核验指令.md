# S1-S3训练数据监督筛选与前向传播核验指令

## 一、核验目标

在正式训练前，检查 S1、S2、S3 当前的数据生成、DataLoader 和训练循环是否满足：

```text
有效监督step：
正常前向传播，并参与损失计算

无效监督step：
仍保留在连续序列中完成前向传播，用于更新SNN状态
但该step不参与监督损失和评价指标计算
```

必须沿着以下完整链路核验：

```text
H5生成
→ Dataset读取
→ block组装
→ 模型forward
→ loss计算
→ backward/optimizer
→ validation/test指标
```

本任务只做核验与报告，不启动正式训练。

---

## 二、统一定义

### 2.1 样本级监督有效性

Car数据应确认采用：

```text
speed_valid = ||T|| > 1.3 m/s
omega_valid = ||Omega|| < 0.18 rad/s
pose_valid = 起止位姿和插值均有效
```

最终：

```text
supervise_valid =
pose_valid
AND speed_valid
AND omega_valid
```

必须从S1、S2、S3实际H5和源码中确认，不能默认三者一致。

### 2.2 像素级mask

像素级mask应独立于样本级筛选：

```text
valid_ttc_mask =
depth_valid
AND reprojection_valid
AND event_active
AND finite(inverse_ttc)
```

训练signed inverse TTC时，不应加入：

```text
Tz > 0
inverse_ttc > 0
```

正TTC区域另存：

```text
positive_ttc_mask =
valid_ttc_mask
AND inverse_ttc > 0
```

### 2.3 最终参与损失的条件

一个像素参与损失，应同时满足：

```text
supervise_valid == 1
AND valid_ttc_mask(x,y) == 1
```

建议显式构造：

```python
loss_mask = (
    supervise_valid[:, None, None]
    & valid_ttc_mask
)
```

并记录：

```text
direct_supervision_valid =
supervise_valid
AND valid_ttc_mask中至少存在一个有效像素
```

## 三、核心行为核验

### 3.1 无效监督step是否仍进入forward

检查是否存在以下错误：

```text
生成H5时直接删除supervise_valid=0的step
Dataset中跳过supervise_valid=0的step
组block前只保留有效监督step
forward前对无效监督step执行continue
```

正确行为：

```text
无效监督step仍保留原时间顺序
仍生成事件输入
仍进入model.forward()
仍更新SNN膜电位和隐藏状态
仅在loss中权重为0
```

### 3.2 10步block是否完整前向

对于每个10步block，应执行：

```python
for t in range(10):
    pred_t = model(event_t)
```

无论 `supervise_valid[t]` 是0还是1，都必须forward。

损失只累计有效监督：

```python
if supervise_valid[t] and valid_ttc_mask[t].any():
    loss_sum += masked_loss(
        pred_t,
        gt_t,
        valid_ttc_mask[t],
    )
    valid_step_count += 1
```

禁止在forward前执行：

```python
if not supervise_valid[t]:
    continue
```

### 3.3 loss归一化

检查当前loss采用哪一种：

```text
按有效像素归一化
按有效step归一化
固定除以10
```

推荐按有效像素，或先做每步masked mean后再按有效step平均。

禁止固定除以10，因为不同block的有效监督密度不同。

### 3.4 全block无监督时

当：

```text
valid_step_count == 0
```

应当：

```text
完成10步forward
更新SNN状态
不执行backward
不执行optimizer.step()
block末尾正常detach_states()
```

同时确认不会出现：

```text
0除法
NaN loss
零损失optimizer.step
错误reset状态
删除block导致时序不连续
```

## 四、SNN状态管理

### 连续时序训练

若block按真实时间顺序输入：

```text
每10步：
backward
optimizer.step
detach_states
```

但不自动：

```text
reset_states
```

只有以下情况reset：

```text
序列开始
timeout造成时间断裂
数据文件切换
显式跳过不连续区间
S3 ROI切换
```

### 随机block训练

若DataLoader对block随机shuffle：

```text
每个block开始必须reset_states
```

因为相邻batch不再连续。

分别说明S1、S2、S3当前采用哪种模式。

## 五、分别核验S1、S2、S3

### S1

确认：

```text
全部step是否保留
supervise_valid=0是否仍有event_cnt
无效step是否进入forward
loss是否正确屏蔽
timeout后是否reset
```

### S2

除上述内容外，确认：

```text
supervise_valid使用S2自身[t_start,t_end]计算的T/Omega
不是沿用S1或S3
valid_ttc_mask来自S2目标几何
不是池化S1最终mask
```

### S3

确认：

```text
supervise_valid使用S3自身ROI固定N时间窗
ROI切换处reset_required=1
无效监督step没有被提前删除
ROI切换前后的状态没有错误继承
```

## 六、统计核验

对S1、S2、S3分别输出：

```text
总step数
supervise_valid=1数量及比例
supervise_valid=0数量及比例
valid_ttc_mask为空的step数量及比例
direct_supervision_valid数量及比例
全部10步有效block比例
部分有效block比例
零有效监督block比例
实际进入forward的step数
实际参与loss的step数
实际执行optimizer.step的block数
```

必须满足：

```text
进入forward的step数
≈
DataLoader读取的全部step数
```

以及：

```text
参与loss的step数
=
direct_supervision_valid的step数
```

不相等时，定位具体代码路径。

## 七、最小运行时验证

分别从S1、S2、S3抽取至少20个连续10步block，覆盖：

```text
10步全部有效
部分step有效
10步全部无效
timeout前后
S3 ROI切换前后
```

训练循环中临时记录：

```text
scheme
block_index
step_in_block
supervise_valid
mask_pixel_count
forward_called
loss_included
reset_called
detach_called
optimizer_step_called
```

重点验证：

```text
supervise_valid=0时：
forward_called=True
loss_included=False

零监督block：
optimizer_step_called=False
```

## 八、禁止行为

发现以下任一种情况必须修正：

```text
生成数据时删除无效监督step
DataLoader只加载supervise_valid=1
forward前跳过无效监督step
用supervise_valid清空事件输入
固定除以10
零监督block仍optimizer.step
将speed/omega条件混入像素mask
S1/S2/S3使用不同筛选口径
```

## 十、最终必须回答

1. S1、S2、S3是否都保留 `supervise_valid=0` 的step；
2. 无效监督step是否确实执行forward；
3. 无效监督step是否完全不参与loss；
4. 像素mask与样本级supervise_valid是否分离；
5. 零监督block是否会错误执行backward或optimizer.step；
6. loss按有效像素、有效step还是固定10步归一化；
7. S1、S2、S3筛选阈值和逻辑是否一致；
8. timeout、序列断裂和S3 ROI切换处是否正确reset；
9. 当前训练数据是否满足“连续前向、稀疏监督”；
10. 若不满足，具体修改哪些文件、函数和代码行。

完成核验和报告后停止，不启动正式训练。
